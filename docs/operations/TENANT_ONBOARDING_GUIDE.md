# Tenant Onboarding Guide

## Scope
Operational onboarding flow for new hotel/pousada tenants in Guest Guide V2.

## 1) Tenant Creation
1. Create organization in `public.organizations` (via Host Connect backoffice flow).
2. Create property in `public.properties` linked to the organization.
3. Record generated `property_id` and `org_id` for operational use.

## 2) property_id Generation and Verification
- `property_id` is UUID generated by the database at property creation.
- Validate mapping:
```sql
select id as property_id, org_id, name
from public.properties
where id = '<PROPERTY_ID>';
```

## 3) Seed Execution
1. Execute tenant-specific seed scripts for `guest_guide` tables.
2. Ensure all seeded rows contain `property_id` and `org_id` consistent with the property.
3. Validate no cross-tenant drift:
```sql
select p.id
from guest_guide.pages p
join public.properties pr on pr.id = p.property_id
where p.property_id = '<PROPERTY_ID>'
  and p.org_id <> pr.org_id;
```

## 4) Content Population
Populate and review:
- `home_configs`
- `pages`
- `page_routes`
- `content_blocks`
- `navigation_nodes`
- `contacts`
- `top_sticker_messages`

## 5) Navigation Setup
- Ensure parent index pages and child pages are correctly linked (`pages.parent_id`).
- Ensure route slugs in `page_routes` map to valid page ids.

## 6) Background Video Setup
- Insert at least one active background video for property.
- Keep one default active video for fallback (`is_default = true`).
- Validate via edge `get_home_config` that background video resolves.

## 7) Partner Onboarding
Populate:
- `partners`
- `partner_schedules`
- `partner_promotions`
- `partner_media` (if used)

Validate via `list_partners` endpoint using tenant header.

## 8) Analytics Validation
1. Emit test events using `track_event`.
2. Verify rows in `guest_guide.event_log` with expected `property_id`/`org_id`.
3. Validate KPI summary endpoint:
- `get_kpi_summary` returns expected structure.

## 9) Operational Smoke
Run:
```bash
npm run ops:smoke:staging
```

Then run explicit KPI check for the same `x-property-id`.

## 10) Readiness Criteria
Tenant is onboarded when:
- seed/content/navigation validated
- background video/stickers validated
- partners visible
- analytics and KPI endpoints returning tenant-scoped data
- QA gates passing in current release commit
